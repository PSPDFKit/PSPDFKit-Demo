Overview

The main view controller to display PDFs. Can be displayed in fullscreen or embedded. Everything in PSPDFKit is based around PSPDFViewController. This is the class you want to override and customize.

Make sure to correctly use viewController containment when adding this as a child view controller. If you override this class, ensure all UIViewController methods you’re using do call super. (e.g. viewWillAppear).

Tasks

Initialization

  • – initWithDocument:

    Initialize with a document. Document can be nil. In this case, just the background is displayed and the HUD stays visible. Also supports creation via initWithCoder to allow usage in Storyboards.

Page Scrolling and Zooming

  •   page

    Current page displayed, not landscape corrected. To change page, use scrollToPage. e.g. if you have 50 pages, you get 25/26 “double pages” when in double page mode.

         property
  •   realPage

    Current page displayed, landscape corrected. To change page, use scrollToPage. This represents the pages in the PDF document, starting at 0.

         property
  • – setPage:animated:

    Control currently displayed page. Page starts at 0.
  • – scrollToNextPageAnimated:

    Scroll to next page. Will potentiall advance two pages in dualPage mode.
  • – scrollToPreviousPageAnimated:

    Scroll to previous page. Will potentiall decrease two pages in dualPage mode.
  • – scrollRectToVisible:animated:

    Scrolls to a specific rect on the current page. No effect if zoom is at 1.0. Note that rect are screen coordinates. If you want to use PDF coordinates, convert them via: PSPDFConvertPDFRectToViewRect() or -convertPDFPointToViewPoint of PSPDFPageView.
  • – zoomToRect:animated:

    Zooms to a specific rect, optionally animated.
  • – setZoomScale:animated:

    Zoom to specific scale, optionally animated.
  •   viewState

    Saves the view state into a serializable object. (page/zoom/position/HUD)

         property
  • – reloadData

    Recreates the content view hierarchy. Usually automatically invoked if you change certain properties (like document, pageTransition).
  • – searchForString:animated:

    Opens the PSPDFSearchViewController with the searchString.

Page Rendering

  •   renderContentOpacity

    Change the PDF content opacity. Defaults to 1 (100%)

         property
  •   renderBackgroundColor

    Multiplies a color to the PDF background color. Does not change anything if the pge color is black. This is useful to ease reading on mostly white documents. Defaults to [UIColor whiteColor]. (white) Can not be set to nil.

         property
  •   renderInvertEnabled

    Inverts PDF rendering. Useful for people that need additional accessibility.

         property
  •   renderAnnotationTypes

    Set what annotations should be rendered. Defaults to PSPDFAnnotationTypeAll. Set this to PSPDFAnnotationTypeLink | PSPDFAnnotationTypeHighlight for PSPDFKit v1 rendering style.

         property

HUD Controls

  •   HUDView

    View that is displayed as HUD. Make a KVO on viewMode if you build a different HUD for thumbnails view. HUDView is created in viewDidLoad. Subclass or use KVO to add your custom views when this changes.

         property
  •   HUDViewMode

    Manages the show/hide mode of the HUD view. Defaults to PSPDFHUDViewAutomatic. Note: this does not affect manually setting HUDVisible. If your statusbar setting is set to PSPDFStatusBarDefault; the HUD will be non-opaque and thus stay visible always. HUD will not change when changing this mode. Use setHUDVisible:animated:.

         property
  •   HUDVisible

    Show or hide HUD controls, titlebar, status bar. (iPhone only)

         property
  • – showControls

    Show the HUD. Respects HUDViewMode.
  • – hideControls

    Hide the HUD. Respects HUDViewMode.
  • – hideControlsAndPageElements

    Hide the HUD (respects HUDViewMode) and additional elements like page selection.
  • – toggleControls

    Toggles the HUD. Respects HUDViewMode.
  •   toolbarEnabled

    Enables default header toolbar. Only displayed if inside UINavigationController. Defaults to YES. Set before loading view.

         property
  •   scrobbleBarEnabled

    Enables bottom scrobble bar [if HUD is displayed]. will be hidden automatically when in thumbnail mode. Defaults to YES. Animatable. There’s some more logic involved, e.g. is the default white statusbar not hidden on a HUD change.

         property
  •   positionViewEnabled

    Enables/Disables the bottom document site position overlay. Defaults to YES. Animatable. Will be added to the hudView.

         property
  •   documentLabelEnabled

    Enable/disable the top document label overlay. Defaults to YES on iPhone and NO on iPad. (On iPad, there’s enough space to show the title in the navigationBar)

         property
  •   renderAnimationEnabled

    If YES, shows a decent UIActivityIndicator on the top right while page is rendering. Defaults to YES.

         property
  •   contentView

    Content view. Use this if you want to add any always-visible UI elements. Created in viewDidLoad. contentView is behind hudView but always visible. ContentView does NOT overlay the navigationBar/statusBar, even if that one is transparent.

         property

Margin is extra space for your (always visible) UI elements.
Content will be moved accordingly.
Area outside margin does not receive touch events, or is shown while zooming.

Padding is space that is displayed around the document.

    property

    If set to YES, tries to find the text blocks on the page and zooms into the tapped block. NO will perform a generic zoom into the tap area. Defauts to YES.

         property

    Enable/disable scrolling. can be used in special cases where scrolling is turned of (temporarily). Defaults to YES.

         property

    Locks the view and the HUD. Disables scrolling, zooming and gestures that would invoke scrolling/zooming. This is useful if you want to invoke a “drawing mode”. (e.g. Ink Annotation drawing) This also blocks programatically calls to scrollToPage. Note: This might be disabled after a reloadData.

         property

    Locks the current set rotation. Defaults to NO. If set to false, it invokes a attemptRotationToDeviceOrientation (iOS5 and above)

         property
  •   scrollOnTapPageEndEnabled

    Tap on begin/end of page scrolls to previous/next page. Defaults to YES.

         property
  •   scrollOnTapPageEndMargin

    Margin at which the scroll to next/previous tap should be invoked. Defaults to 60.

         property
  •   textSelectionEnabled 
    Allows text selection. Defaults to YES.
    property
  •   passwordDialogEnabled

    If YES, when a PDF that requires a password is set, a password dialog is shown. (The password dialog is of class PSPDFPasswordView; customize with overrideClassNames) Defaults to YES. If NO, an attempt to display the document anyway is made.

         property
  •   useParentNavigationBar

    If embedded via iOS5 viewController containment, set this to true to allow this controller to access the parent navigationBar/navigationController to add custom buttons. Has no effect if toolbarEnabled is false or there’s no parentViewController. Defaults to NO.

         property
  •   linkAction

    Set the default link action for pressing on PSPDFLinkAnnotations. Default is PSPDFLinkActionInlineBrowser. Note: if modal is set in the link, this property has no effect.

         property

Appearance Properties

  •   pageMode

    Set a PageMode defined in the enum. (Single/Double Pages) Reloads the view, unless it is set while rotation is active. Thus, one can customize the rotation behavior with animations when set within willAnimate*. Defaults to PSPDFPageModeAutomatic on iPad and PSPDFPageModeSingle on iPhone.

         property
  •   pageTransition

    Defines the page transition. Replaces pageCurlEnabled; allows more modes.

         property
  •   pageScrolling

    Change scrolling direction. defaults to horizontal scrolling. (PSPDFScrollDirectionHorizontal) Only relevant for scrolling page transitions.

         property
  •   doublePageModeOnFirstPage

    Shows first document page alone. Not relevant in PSPDFPageModeSinge. Defaults to NO.

         property
  •   zoomingSmallDocumentsEnabled

    Allow zooming of small documents to screen width/height. Defaults to YES.

         property
  •   pageCurlDirectionLeftToRight

    For Left-To-Right documents, this sets the pagecurl to go backwards. Defaults to NO. Note: doesn’t re-order document pages. There’s currently no real LTR support in PSPDFKit.

         property
  •   fitToWidthEnabled

    If true, pages are fit to screen width, not to either height or width (which one is larger – usually height.) Defaults to NO. iPhone switches to yes in willRotateToInterfaceOrientation – reset back to no if you don’t want this. fitToWidthEnabled is not supported for vertical scrolling or pageCurl mode. (You might want PSPDFPageScrollContinuousTransition)

         property
  •   fixedVerticalPositionForfitToWidthEnabledMode

    Defaults to NO. If this is set to YES, the page remembers its vertical position if fitToWidthEnabled is enabled. If NO, new pages will start at the page top position.

         property
  •   clipToPageBoundaries

    PageCurl mode only: clips the page to its boundaries, not showing a pageCurl on empty background. Defaults to YES. Usually you want this, unless your document is variable sized.

         property
  •   minimumZoomScale

    Minimum zoom scale. Defaults to 1. You usually don’t want to change this.

         property
  •   maximumZoomScale

    Maximum zoom scale for the scrollview. Defaults to 10. Set before creating the view.

         property
  •   pagePadding

    Page padding width between single/double pages. Defaults to 20.

         property
  •   shadowEnabled

    Enable/disable page shadow.

         property
  •   statusBarStyleSetting

    Status bar styling. Defaults to PSPDFStatusBarSmartBlack. If controller is used embedded (in a non-fullscreen way), this setting has no effect.

         property
  •   backgroundColor

    If not set, we’ll use scrollViewTexturedBackgroundColor as default.

         property
  •   tintColor

    Set global toolbar tint color. Overrides defaults. Default is nil (depends on statusBarStyleSetting)

         property
  •   shouldTintPopovers

    Enable to add tinting to UIPopoverController. (using a custom UIPopoverView subclass) iOS5 and later. Defaults to YES. New since PSPDFKit 2.2.

         property
  •   navigationBarHidden

    The navigationBar is animated. Check this to get the proper value, even if navigationBar.navigationBarHidden is not yet set (but will be in the animation block)

         property
  •   annotationAnimationDuration

    Annotations are faded in. Set global duration for this fade here. Defaults to 0.25f.

         property

Helpers

  • – isDoublePageMode

    Depending on pageMode, this returns true if two pages are displayed.
  • – isRightPageInDoublePageMode:

    Checks if the current page is on the right side, when in double page mode. Page starts at 0.
  • – presentModalViewController:embeddedInNavigationController:withCloseButton:animated:

    Show a modal view controller with automatically added close button on the left side.
  • – presentViewControllerModalOrPopover:embeddedInNavigationController:withCloseButton:animated:sender:options:

    Show a modal view controller or a popover with automatically added close button on the left side. Use sender OR rect (both only needed for the popover)
  • – visiblePageNumbers

    Return an NSNumber-array of currently visible page numbers.
  • – visiblePageViews

    Return array of currently visible PSPDFPageView’s.
  • – isLastPage

    YES if we are at the last page.
  • – isFirstPage

    YES if we are at the first page.
  • – masterViewController

    Returns the topmost, active viewcontroller.

SubclassingHooks Methods

  •   overrideClassNames

    Use this to use specific subclass names instead of the default PSPDF* classes. e.g. add an entry of [PSPDFPageView class] / [MyCustomPageView class] as key/value pair to use the custom subclass. (MyCustomPageView must be a subclass of PSPDFPageView) Throws an exception if the overriding class is not a subclass of the overridden class.

         property
  • – createToolbarAnimated:

    Override if you’re changing the toolbar to your own. The toolbar is only displayed, if PSPDFViewController is inside a UINavigationController.
  • – updateBarButtonItem:animated:

    Request to update a specific barButton. Might be more efficent than using createToolbar.
  • – updateGridForOrientation

    Can be subclassed to update grid spacing.
  • – updateSettingsForRotation:

    Called in viewWillAppear with the initial rotation and then in willRotateToInterfaceOrientation. Might be called multiple times during a rotation, so any code in there should be fast. The default implementation for this is empty.
  • – statusBarStyle

    Manually return the desired UI status bar style (default is evaluated via app status bar style)
  •   pageTransitionController

    Access internal UIViewController for displaying the PDF content

         property
  • – applicationDidEnterBackground:

    Default saves annotations when app goes to background. Is registered on init/deregistered on dealloc. Only tries to saves document if view is visible.

Internal Methods

  • – handleTouchUpForAnnotationIgnoredByDelegate:

    Causes the annotations to be handled as the receiver sees fit (for example, link annotations are followed).

@property (nonatomic, strong) NSArray *leftBarButtonItems

Discussion

Note that it appears that UIKit limits the left toolbar size if spaces is low in the toolbar, potentially cutting off buttons in those toolbars if the title is also too long. You can either reduce the number of buttons, cut down the text or use a titleView to fix this problem. It also appears that UIKit focues on the leftToolbar, the right one is cut of much later. This problem only appears on the iPad in portrait mode. You can also use updateSettingsForRotation to adapt the toolbar for portrait/landscape mode.

Defines the page transition. Replaces pageCurlEnabled; allows more modes.

@property (nonatomic, assign) PSPDFPageTransition pageTransition

Discussion

Note about PSPDFPageCurlTransition: PageCurl needs iOS5 and above and will fall back to default scrolling on iOS4. PageCurl is more memory intensive; you might wanna disable this on an iPad1. (e.g. with using the PSPSDIsCrappyDevice() to check for modern devices)

If you change the property dynamically depending on the screen orientation, don’t use willRotateToInterfaceOrientation but didRotateFromInterfaceOrientation, else the controller will get in an invalid state.

Returns the topmost, active viewcontroller.

Discussion

This tries to be smart and even works in weird, non-default situations where viewControllers are embedded w/o iOS5 child controller embedding.

If you get effects like the email controller not appearing at all, override this and return the controller where modal controllers can be pushed onto. (Try “return self” first)

It’s a sad thing that this tends to be one of the most complex things in iOS development to get right.
