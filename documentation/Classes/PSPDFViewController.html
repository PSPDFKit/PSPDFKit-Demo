PSPDFViewController Class Reference

Inherits from PSPDFBaseViewController : UIViewController
Conforms to MFMailComposeViewControllerDelegate
PSPDFSearchDelegate
UIPopoverControllerDelegate
UIScrollViewDelegate
Declared in PSPDFViewController.h

Overview

The main view controller to display pdfs. Can be displayed in fullscreen or embedded. When embedded, be sure to correctly relay the viewController calls of viewWillAppear/etc. (or use iOS5 view controller containment)

Tasks

Initialization

  • – initWithDocument:

    Initialize with a document.

Page Scrolling and Zooming

  • – scrollToPage:animated:

    Control currently displayed page. Page starts at 0.
  • – scrollToPage:animated:hideHUD:

    Control currently displayed page, optionally show/hide the HUD. Page starts at 0.
  • – scrollToNextPageAnimated:

    Scroll to next page.
  • – scrollToPreviousPageAnimated:

    Scroll to previous page.
  • – scrollRectToVisible:animated:

    Scrolls to a specific rect on the current page. No effect if zoom is at 1.0. Note that rect are screen coordinates. If you want to use pdf coordinates, convert them via: PSPDFConvertPDFRectToViewRect() or -convertPDFPointToViewPoint of PSPDFPageView.
  • – zoomToRect:animated:

    Zooms to a specific rect, optionally animated.
  • – documentViewState

    Save view state (page/zoom/position)
  • – restoreDocumentViewState:animated:

    Restore view state (page/zoom/position) Note: restoring a certain zoomscale/rect is currently not animatable.

Reloading Content

  • – reloadData

    Reload scrollview. Call if you manually change the view width/height or some property inside PSPDFDocument. Usually not needed.
  • – reloadDataAndScrollToPage:

    Reload scrollview, scroll to specified page.

Thumbnail View

  •   viewMode

    View mode: PSPDFViewModeDocument or PSPDFViewModeThumbnails.

         property
  •   thumbnailSize

    Change thumbnail size. Default is 170x220.

         property
  •   iPhoneThumbnailSizeReductionFactor

    Thumbnails on iPhone are smaller – you may change the reduction factor. Defaults to 0.5.

         property

Class Accessors

  • – pageViewForPage:

    Return the pageView for a given page. Returns nil if page is not initalized (e.g. page is not visible.) Usually, using the delegates is a better idea to get the current page.
  •   popoverController

    Saves the popoverController if currently displayed.

         property
  •   pagingScrollView

    Paging scroll view. (hosts scollviews for pdf)

         property

Helpers

  • – isDualPageMode

    Depending on pageMode, this returns true if two pages are displayed.
  • – isRightPageInDoublePageMode:

    Checks if the current page is on the right side, when in double page mode. Page starts at 0.
  • – presentModalViewController:withCloseButton:animated:

    Show a modal view controller with automatically added close button on the left side.
  • – visiblePageNumbers

    Return array of currently visible page numbers.
  • – isLastPage

    YES if we are at the last page
  • – isFirstPage

    YES if we are at the first page

HUD Controls

  •   hudView

    View that is displayed as HUD. Make a KVO on viewMode if you build a different HUD for thumbnails view. HUD is created in viewDidLoad. Subclass or use KVO to add your custom views when this changes.

         property
  •   HUDVisible

    Show or hide HUD controls, titlebar, status bar. (iPhone only)

         property
  • – setHUDVisible:animated:

    Animated show or hide HUD controls, titlebar, status bar. (status bar fade is iPhone only)
  •   scrobbleBarEnabled

    Enables bottom scrobble bar [if HUD is displayed]. will be hidden automatically when in thumbnail mode. Defaults to YES. Animatable. There’s some more logic involved, e.g. is the default white statusbar not hidden on a HUD change.

         property
  •   positionViewEnabled

    Enables/Disables the bottom document site position overlay. Defaults to YES. Animatable. Will be added to the hudView.

         property
  •   toolbarEnabled

    Enables default header toolbar. Only displayed if inside UINavigationController. Defaults to YES. Set before loading view.

         property

Appearance Properties

  •   pageMode

    Set a PageMode defined in the enum. (Single/Double Pages) Reloads the view, unless it is set while rotation is active. Thus, one can customize the rotation behavior with animations when set within willAnimate*.

         property
  •   pageScrolling

    Change scrolling direction. defaults to horizontal scrolling. (PSPDFScrollingHorizontal)

         property
  •   doublePageModeOnFirstPage

    Shows first document page alone. Not relevant in PSPDFPageModeSinge. Defaults to NO.

         property
  •   zoomingSmallDocumentsEnabled

    Allow zooming of small documents to screen width/height. Defaults to YES.

         property
  •   pageCurlEnabled

    Enables iBooks-like page curl feature. Works only with iOS5 or later. Falls back to default scrolling on iOS4. Defaults to NO. Note: doesn’t work well with non-equal sized documents. Use scrolling if you have such complex documents. Note: You might wanna disable this on the iPad1, because this is more memory hungry than classic scrolling. You can use pageCurlEnabled = !PSPSDIsCrappyDevice(); which will return YES for older devices only. If you change the property dynamically depending on the screen orientation, don’t use willRotateToInterfaceOrientation but didRotateFromInterfaceOrientation, else the controller will get in an invalid state.

         property
  •   pageCurlDirectionLeftToRight

    For Left-To-Right documents, this sets the pagecurl to go backwards. Defaults to NO. Note: doesn’t re-order document pages. There’s currently no real LTR support in PSPDFKit.

         property
  •   fitWidth

    If true, pages are fit to screen width, not to either height or width (which one is larger – usually height.) Defaults to NO. iPhone switches to yes in willRotateToInterfaceOrientation – reset back to no if you don’t want this. fitWidth is currently not supported for vertical scrolling or pageCurl mode.

         property
  •   fixedVerticalPositionForFitWidthMode

    Defaults to NO. If this is set to YES, the page remembers its vertical position if fitWidth is enabled. If NO, new pages will start at the page top position.

         property
  •   clipToPageBoundaries

    PageCurl mode only: clips the page to its boundaries, not showing a pageCurl on empty background. Defaults to YES. Usually you want this, unless your document is variable sized.

         property
  •   maximumZoomScale

    Maximum zoom scale for the scrollview. Defaults to 5.0. Set before creating the view.

         property
  •   pagePadding

    Page padding width between single/double pages. Defaults to 20.

         property
  •   shadowEnabled

    Enable/disable page shadow.

         property
  •   statusBarStyleSetting

    Status bar styling. Defaults to PSPDFStatusBarSmartBlack. If controller is used embedded (in a non-fullscreen way), this setting has no effect.

         property
  •   backgroundColor

    If not set, we’ll use scrollViewTexturedBackgroundColor as default.

         property
  •   tintColor

    Set global toolbar tint color. Overrides defaults. Default is nil (depends on statusBarStyleSetting)

         property
  •   navigationBarHidden

    The navigationBar is animated. Check this to get the proper value, even if navigationBar.navigationBarHidden is not yet set (but will be in the animation block)

         property
  •   annotationAnimationDuration

    Annotations are faded in. Set global duration for this fade here. Defaults to 0.25f.

         property
  •   loadThumbnailsOnMainThread

    Defaults to YES. Set to NO for faster page-scrolling, but may flash white even if there are thumbnails supplied.

         property

Subclassing Helpers

  •   overrideClassNames

    Use this to use specific subclass names instead of the default PSPDF* classes. e.g. add an entry of [PSPDFScrollView class] / [MyCustomPSPDFScrollViewSubclass class] as key/value pair to use the custom subclass.

         property
  •   useParentNavigationBar

    If embedded via iOS5 viewController containment, set this to true to allow this controller to access the parent navigationBar to add custom buttons. Has no effect if toolbarEnabled is false or there’s no parentViewController.

         property
  • – masterViewController

    returns the topmost active viewcontroller. override if you have a custom setup of viewControllers
  • – createToolbar

    override if you’re changing the toolbar to your own. Note: The toolbar is only displayed, if PSPDFViewController is inside a UINavigationController.
  • – contentRect

    Return rect of the content view area excluding translucent toolbar/statusbar.
  •   minLeftToolbarWidth

    UIBarButtonItem doesn’t support calculation of it’s width, so we have to approximate. This allows you to change the minimum width if the heuristics fail. Note: Set this in your subclass within updateToolbars, then call [super updateToolbars].

         property
  • – gridView

    Setup the grid view. Call [super gridView] and modify it to your needs.
  • – updateGridForOrientation

    Can be subclassed to update grid spacing.
  • – statusBarStyle

    Manually return the desired UI status bar style (default is evaluated via app status bar style)

PSPDFInternal Methods

  • – handleTouchUpForAnnotationIgnoredByDelegate:

    causes the annotations to be handled as the receiver sees fit (for example, link annotations are followed).
