PSPDFMenuItem Class Reference

Inherits from UIMenuItem
Declared in PSPDFMenuItem.h

Overview

This subclass adds support for a block-based action on UIMenuItem. If you are as annoyed about the missing target/action pattern, you will love this.

If you use PSPDFMenuItem with the classic initWithTitle:selector initializer, this will work and be handled just like a UIMenuItem.

Warning: By design, PSPDFMenuItem will only work with different title names. Title is required to be > 0 and unique, even when images are used.

Class Methods

installMenuHandlerForObject:

Installs the menu handler. Needs to be called once per class. (A good place is the +load method)

+ (void)installMenuHandlerForObject:(id)object

Discussion

Following methods will be swizzled: – canBecomeFirstResponder (if object doesn’t already return YES) – canPerformAction:withSender: – methodSignatureForSelector: – forwardInvocation:

The original implementation will be called if the PSPDFMenuItem selector is not detected.

@parm object can be an instance or a class.
